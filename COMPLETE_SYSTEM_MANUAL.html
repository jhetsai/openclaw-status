## 1. 系統概述

本系統包含兩個子系統，用於神腦國際門市管理：

| 子系統 | 功能 | 托管平台 |
|--------|------|---------|
| 庫存盤點系統 | 商品庫存追蹤、條碼掃描、LINE照片上傳 | GitHub Pages |
| 台電電費分析系統 | 用電資料視覺化、多維度分析 | GitHub Pages |

## 2. 庫存盤點系統

### 2.1 功能說明

#### 主要功能
- 📸 **LINE 照片上傳**：在 LINE 傳商品照片，AI 自動辨識品號、品名、數量
- 🌐 **網頁即時查看**：透過 GitHub Pages 網頁隨時查看最新庫存
- 📷 **條碼掃描查詢**：用手機相機掃描條碼，快速查詢商品資料
- ✅ **勾選確認**：可標記已盤點的商品
- 📝 **備註記錄**：可為每項商品新增備註
- 🔄 **累積型資料**：每次上傳新增或更新，不會覆蓋舊資料

#### 資料格式

```json
{
  "last_updated": "2026-04-04 13:18",
  "categories": [{
    "name": "商品",
    "items": [{
      "code": "091083113133",
      "name": "MEGA KING 三星 A37/A56/A36 滿版玻璃保護貼",
      "quantity": 3,
      "onlineQuantity": 0,
      "verified": true,
      "note": "",
      "lastUploaded": "2026-04-04 11:28"
    }]
  }]
}
```

### 2.2 連結資訊

| 項目 | 連結 |
|------|------|
| 庫存網頁 | https://jhetsai.github.io/inventory/ |
| GitHub 倉庫 | https://github.com/jhetsai/inventory |

### 2.3 LINE Bot 運作流程

```
[LINE 照片] → [LINE Bot 接收] → [AI Vision 分析] → [更新 inventory.json]
                                                            ↓
                                                   [Git Hook 自動推送]
                                                            ↓
                                                   [GitHub Pages 更新]
                                                            ↓
                                                   [網頁顯示]
```

### 2.4 網頁功能模組

| 模組 | 說明 |
|------|------|
| 統計卡片 | 顯示品項數、實際總數、網購總數、有庫存、零庫存 |
| 搜尋功能 | 輸入品號或品名，即時過濾顯示結果 |
| 條碼掃描 | 開啟相機掃描，快速查詢商品完整資料 |
| 確認模式 | 勾選已盤點商品，可輸入備註 |
| 最近上傳 | 只顯示最近一次上傳的品項 |

### 2.5 欄位規格

| 欄位 | 寬度 | 說明 |
|------|------|------|
| 品號 | 95px | 條碼數字，不自動換行 |
| 品名 | 100px | 商品名稱，自動換行 |
| 實際 | 35px | 實體店面庫存 |
| 網購 | 35px | 線上通路庫存 |
| ✓ | 30px | 確認狀態勾選 |
| 備註 | 自動 | 自由填寫 |

## 3. 台電電費分析系統

### 3.1 功能說明

#### 主要功能
- 📊 **用電量圖表**：柱狀圖顯示各期用電
- 💰 **電費分析**：圖表顯示各期電費
- 🔍 **多維度篩選**：按電號、年度、期別篩選
- 📋 **詳細資料表**：顯示每期明細
- 📱 **響應式設計**：支援電腦、平板、手機

### 3.2 連結資訊

| 項目 | 連結 |
|------|------|
| 分析網頁 | https://jhetsai.github.io/openclaw/electricity_final.html |
| GitHub 倉庫 | https://github.com/jhetsai/openclaw |

### 3.3 資料格式

```json
{
  "19-51-2353-20-8": {
    "t": "住宅",
    "a": "水林鄉西井段2568地號",
    "f": "XG31"
  }
}
```

用電記錄：
```json
{a: "19-51-2353-20-8", p: "11311", y: 2024, k: 225, c: 368}
```

| 欄位 | 說明 |
|------|------|
| a | 電號 |
| p | 期別代碼（11311 = 113年11月） |
| y | 西元年份 |
| k | 用電度數 |
| c | 電費金額 |

### 3.4 視圖模式

| 模式 | 說明 |
|------|------|
| 依年度 | 合併同一年度的所有期別 |
| 依期別(每2月) | 顯示每期用電/電費柱狀圖 |
| 指定期別各電號 | 顯示該期各電號的用電比較表 |

## 6. 部署與維護

### 6.1 LINE Bot Webhook 更新

當 ngrok 重啟後，URL 會改變：

1. 前往 LINE Developers Console
2. 進入你的 Channel
3. Messaging API settings
4. 更新 Webhook URL
5. 點擊 Verify

### 6.2 常見問題

| 問題 | 解決方式 |
|------|---------|
| 網頁 404 | 等待 GitHub Pages 部署（15-30秒） |
| LINE Bot 無回應 | 檢查 ngrok 是否運行 |
| 條碼掃描失敗 | 確保光線充足，相機權限開啟 |

## 7. 操作指南

### 7.1 庫存系統 - LINE 操作

#### 上傳庫存照片
1. 打開 LINE，進入 Bot 對話
2. 拍攝商品照片（盡量清晰）
3. 傳照片並說「**我要盤點**」
4. Bot 回覆「已更新庫存，共 X 項商品」

### 7.2 庫存系統 - 網頁操作

#### 搜尋商品
1. 在搜尋框輸入品號或品名
2. 系統即時過濾顯示結果

#### 條碼掃描查詢
1. 點擊「📷 查詢」
2. 允許相機權限
3. 對準條碼掃描
4. 顯示商品完整資料

#### 確認盤點
1. 點擊「✏️ 確認」
2. 勾選已盤點的商品
3. 可輸入備註
4. 點擊「✓ 完成」

### 7.3 台電系統 - 操作

#### 查看用電概況
1. 開啟網頁
2. 預設顯示全部電號的總計

#### 篩選特定電號
1. 在第一個下拉選單選擇電號
2. 圖表和資料自動更新
