## 🖥️ 系統狀態頁面 Bake-in 方案（2026-04-11 下午）

### 問題背景
- R2 上的系統狀態頁面（`openclaw-status/index.html`）試圖 fetch `localhost:3838` 和 `api.open-meteo.com`
- 從瀏覽器打開 R2 網址時，兩者都被 CORS 阻擋（特別是 R2 本身沒有 CORS header）
- 導致「系統概覽」欄位（主機/資源/天氣）全部空白

### 解決方案：Bake-in
- 由心跳 cron 定期執行 `bake-status-page.sh`（新增）
- 腳本 fetch API → 把資料直接寫入 HTML → 上傳 R2
- 頁面打開時直接顯示已 bake 的資料，**不需要 fetch**

### 新增腳本
| 檔案 | 功能 |
|------|------|
| `scripts/bake-status-page.sh` | 主腳本，協調 fetch + bake + upload |
| `scripts/bake-status-page.py` | Python，執行字串替換（bake 資料進 HTML） |

### 修正的 Bug
1. **TS 變數在 heredoc 展開問題**：原本嘗試用 `$(date)` 在 bash heredoc 傳給 Python，但 TS 在 Python 才取最新時間
   - 解決：Python 自己用 `datetime.now()` 取 TS
2. **正則引號 mismatch**：正則用單引號 `class='label'`，HTML 是雙引號 `class="label"`
   - 解決：正則改用 `class="label"`（無引號敏感）
3. **fmt(now) 只替換第一個**：一開始用 `c.replace("var ts = fmt(now);", ...)` 只換第一處
   - 解決：用 `re.sub(r"var ts = '[^']*';", ...)` 替換整行

### Bake 的欄位
- 開機時間、記憶體（used/total）、CPU 使用率（來自 `localhost:3838/api/status`）
- 氣溫、風速、天氣（來自 `api.open-meteo.com`）
- 版本時間（Python `datetime.now()`）

### 標籤修正
- `📡 最後刷新時間` → `📡 版本最後上傳時間`
- 移除 footer（`蝦助出品`）

### R2 HTML 與 GitHub HTML 的差異
- **GitHub**：`update-status-page.sh`（心跳執行）→ bake + push GitHub Pages
- **R2**：`bake-status-page.sh`（需手動或另設 cron）→ bake + 上傳 R2
- 兩個腳本各自維護自己的 HTML

### 依賴關係
- `bake-status-page.sh` → `bake-status-page.py`
- Python 依賴：`boto3`（已安裝）
- 網路依賴：`localhost:3838` + `api.open-meteo.com`
