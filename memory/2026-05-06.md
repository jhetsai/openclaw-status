## 台股股息殖利率表格 A1 修正（11:00-11:30）

### 問題根因
用戶反覆說「0056 的 42.61 數值錯誤」：

`update_dividend_data.py` 的 `tw_dividend_detail.json` parse 邏輯有問題——
爬蟲從 Yahoo 抓回來的 `cash_dividend: 42.61` 並不是「每股股利」，而是**除息日的前一天收盤價**（昨收價）。

實際驗證：
- `confirmed_tw`（已除息）：0056 2025Q4 → cash = 0.866 元/股 ✅
- `pending_tw`（待發放）：0056 2026Q1 → cash = 1.0 元/股 ✅
- `tw_dividend_detail.json`：0056 2026Q1 → cash_dividend = 42.61（錯誤）

0.866 × 6000 股 = 5,196 元（符合 confirmed_tw 的 amount）
1.0 × 6000 股 = 6,000 元（符合 pending_tw 的 amount）

所以 `confirmed_tw` / `pending_tw` 裡的 `cash` 欄位才是正確的「每股股利」。

### 修復內容
修改 `update_dividend_data.py` 的 `tw_div_info_computed` 段落：
- **移除**從 `tw_dividend_detail.json` 取 `cash_dividend` 的邏輯（該檔案數值錯誤）
- **改用** `confirmed_tw` 和 `pending_tw` 的 `cash` 欄位（已確認正確的每股股利）
- 掃描所有 pending_tw → 取每檔最新一筆 → 作為 `div`
- 年化 ann_div = div × per_year（季配×4，半年配×2，月配×12）

修正後 0056：
- `div`: 1.0 元/股
- `freq`: 季配
- `ann_div`: 4.0（年化）
