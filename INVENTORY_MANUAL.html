## 1. 系統簡介

本系統用於**神腦國際門市庫存管理**，幫助追蹤商品數量和盤點狀態。

**特色功能：**
- 📸 LINE 照片上傳，AI 自動辨識商品
- 🌐 網頁即時查看庫存
- 📷 條碼掃描查詢
- ✅ 勾選確認已盤點商品
- 📝 備註記錄功能

## 4. LINE 照片上傳流程

### 4.1 基本操作

1. 打開 LINE，進入「庫存管理 Bot」
2. 拍攝商品照片（盡量清晰）
3. 傳照片給 Bot，並說「**我要盤點**」
4. Bot 自動分析照片，更新資料庫

### 4.2 Bot 自動執行的動作

```
1. AI 視覺辨識照片中的商品
2. 比對品號（條碼）
3. 更新數量（實際庫存、網購庫存）
4. 標記 lastUploaded 時間
5. 自動同步到 GitHub
```

### 4.3 注意事項

- 📷 照片越清晰，辨識越準確
- 📦 每次上傳會增量更新（舊商品不消失）
- ✅ 已存在的商品會更新數量
- 🆕 新商品會自動加入資料庫
- 🔄 無需重複上傳相同的商品

### 5.3 功能說明

#### 🔍 搜尋功能
- 輸入品號或品名
- 即時過濾顯示結果

#### 📷 掃描查詢
1. 點擊「📷 查詢」
2. 開啟相機，掃描商品條碼
3. 顯示商品完整資料（品號、品名、數量、最後上傳時間）

#### ✏️ 確認模式
1. 點擊「✏️ 確認」
2. 出現勾選框
3. 勾選已盤點的商品
4. 可輸入備註
5. 點擊「✓ 完成」儲存

### 5.4 狀態標記

| 符號 | 意義 |
|------|------|
| ✅ | 已確認（已盤點） |
| ⏳ | 待確認（尚未盤點） |
| 綠色背景 | 已確認列 |

## 6. 欄位說明

| 欄位 | 說明 |
|------|------|
| 品號 | 商品的唯一識別碼（條碼） |
| 品名 | 商品的名稱 |
| 實際 | 實體店面庫存數量 |
| 網購 | 網路通路庫存數量 |
| ✓ | 是否已確認盤點 |
| 備註 | 額外備註資訊 |

## 7. 常見問題

### Q1: 網頁顯示 404？
可能是 GitHub Pages 部署延遲，等待 15-30 秒後再試。

### Q2: LINE Bot 無回應？
檢查網路連線，或過幾分鐘後再試。

### Q3: 掃描失敗？
- 確保光線充足
- 對準條碼
- 保持手機穩定

### Q4: 資料不同步？
告訴我，我幫你檢查和手動同步。
