🄀 Synology DS220+ 安裝 MinIO 教學

DSM 圖形化操作，無需指令列

📋 前置準備

  • Synology DS220+（RAM 已擴充到 10GB）✅
  • DSM 7.x 版本
  • 一個共用的資料夾（用來存放 MinIO 資料）

步驟一：安裝 Docker 套件

1
開啟套件中心
2
搜尋「Docker」→ 點「安裝」
3
安裝完成後，DSM 左側找到Docker圖示，點擊讓服務啟動

Docker 必須保持運行，MinIO 才會在背景執行

步驟二：建立 MinIO 專用資料夾

1
開啟File Station
2
在共用資料夾（例如 docker 或 public）裡建立新資料夾：
minio → data
minio → config
3
在每個資料夾上按右鍵 → 「內容」→ 「權限」→
新增「everyone」→ 權限勾選「完全控制」→ 儲存

步驟三：下載並啟動 MinIO 容器（DSM 圖形）

1
開啟 DSM 左側的Docker套件
2
點左側「註冊表」→ 右上搜尋「minio」→ 找到官方 minio/minio → 點「選擇」→「下載」

映像檔約 200MB，下載需要幾分鐘

3
下載完成後，點左側「映像檔」→ 找到 minio → 點「啟動」
📷 DSM 截圖：Docker → 註冊表 → 搜尋 minio → 下載完成 → 啟動

啟動設定（重要！一個一個填）

A
容器名稱：填 minio
B
新增連接埠轉遞：
本機連接埠 9000 → 容器連接埠 9000
本機連接埠 9001 → 容器連接埠 9001

兩個都要新增！

C
新增環境變數（Environment Variables）：
MINIO_ROOT_USER = minioadmin
MINIO_ROOT_PASSWORD = 你的密碼（至少8字）

請記下你設定的密碼

D
新增資料夾掛載：
本機資料夾 → 選取 /volume1/docker/minio/data
掛載路徑 → 填 /data
（再新增一筆）
本機資料夾 → 選取 /volume1/docker/minio/config
掛載路徑 → 填 /root/.minio
E
啟動策略：勾選「啟用自動重新啟動」
F
點「完成」啟動容器

步驟四：開通 NAS 防火牆連接埠

1
DSM 控制台 → 安全性 → 防火牆
2
點「編輯防火牆規則」
3
新增規則：
動作 → 允許
連接埠 → 自訂（9000, 9001）
來源 IP → 任意（0.0.0.0/0）
4
儲存套用

步驟五：開網頁設定 Bucket

1
在電腦瀏覽器打開：
http://你的NAS_IP:9001

例如：http://192.168.18.200:9001

2
登入 MinIO 管理介面
帳號：minioadmin
密碼：你剛設定的密碼
3
進入後點左側「Buckets」→「Create Bucket」

命名建議：openclaw 或 shared

4
建立完成後，點左側「Access Keys」→「Create access key」

把Access Key和Secret Key記下來，等一下要給蝦助

✅ 完成！請告訴蝦助以下資訊

請提供這 4 個資訊：
  1. NAS IP 位址（例如 192.168.18.200）
  2. Bucket 名稱（例如 openclaw）
  3. Access Key（金鑰 ID）
  4. Secret Key（私密金鑰）

有了這些，我就可以把 R2 上傳改成送到你的 NAS 🚀
