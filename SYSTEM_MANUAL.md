# 2. 庫存盤點系統

## 2.1 功能說明

### 主要功能

- **LINE 照片上傳**：在 LINE 傳商品照片，AI 自動辨識品號、品名、數量
- **網頁即時查看**：透過 GitHub Pages 網頁隨時查看最新庫存
- **條碼掃描查詢**：用手機相機掃描條碼，快速查詢商品資料
- **勾選確認**：可標記已盤點的商品
- **備註記錄**：可為每項商品新增備註
- **累積型資料**：每次上傳新增或更新，不會覆蓋舊資料

### LINE Bot 運作流程

1. 使用者在 LINE 傳照片並說「我要盤點」
2. Bot 接收圖片
3. AI Vision 分析圖片內容
4. AI 回傳 JSON 格式的庫存資料
5. Bot 解析並更新 inventory.json
6. 更新 lastUploaded 時間戳記
7. Git Hook 自動推送至 GitHub
8. GitHub Pages 自動部署更新
9. 網頁顯示最新資料

## 2.2 資料格式

### inventory.json 結構

```json
{
  "last_updated": "2026-04-04 13:30",
  "categories": [
    {
      "name": "商品分類名稱",
      "items": [
        {
          "code": "091083113133",
          "name": "MEGA KING 三星 A37/A56/A36 滿版玻璃保護貼",
          "quantity": 3,
          "onlineQuantity": 0,
          "verified": true,
          "note": "備註文字",
          "lastUploaded": "2026-04-04 11:28"
        }
      ]
    }
  ]
}
```

### 欄位說明

| 欄位 | 類型 | 說明 |
|------|------|------|
| last_updated | string | 資料最後更新時間 |
| categories | array | 商品分類陣列 |
| name | string | 分類名稱 |
| items | array | 該分類下的商品陣列 |
| code | string | 商品品號（條碼） |
| name | string | 商品名稱 |
| quantity | number | 實際庫存數量 |
| onlineQuantity | number | 網購庫存數量 |
| verified | boolean | 是否已確認盤點 |
| note | string | 備註 |
| lastUploaded | string | 最後上傳時間 |

## 2.3 網頁功能模組

### 統計看板

- 本次品項：顯示本次上傳的品項數量
- 實際總數：所有商品的實際庫存總和
- 網購總數：所有商品的網購庫存總和
- 有庫存：實際或網購 > 0 的品項數
- 零庫存：實際=0 且 網購=0 的品項數

### 搜尋功能

- 輸入品號或品名
- 即時過濾顯示結果
- 支援模糊搜尋

### 條碼掃描

- 使用 html5-qrcode 函式庫
- 開啟手機相機掃描條碼
- 顯示商品完整資料（品號、品名、數量、最後上傳時間）

### 確認模式

- 點擊「✏️ 確認」進入編輯模式
- 可勾選已盤點商品
- 可輸入備註
- 點擊「✓ 完成」儲存

## 2.4 顯示邏輯

- 只顯示「最近一次上傳」的品項（24小時內）
- 已存在的商品會更新數量
- 新商品會自動加入
- 零庫存品項（實際=0 且 網購=0）預設勾選且鎖定

# 3. 台電電費分析系統

## 3.1 功能說明

### 主要功能

- **用電量圖表**：柱狀圖顯示各期用電
- **電費分析**：圖表顯示各期電費
- **多維度篩選**：按電號、年度、期別篩選
- **詳細資料表**：顯示每期明細
- **響應式設計**：支援電腦、平板、手機

## 3.2 資料格式

### 電號基本資料

```json
{
  "19-51-2353-20-8": {
    "t": "住宅",
    "a": "水林鄉西井段2568地號",
    "f": "XG31"
  }
}
```

### 用電記錄

```json
{
  "a": "19-51-2353-20-8",
  "p": "11311",
  "y": 2024,
  "k": 225,
  "c": 368
}
```

### 欄位說明

| 欄位 | 說明 | 範例 |
|------|------|------|
| a | 電號 | 19-51-2353-20-8 |
| p | 期別代碼 | 11311（113年11月） |
| y | 西元年份 | 2024 |
| k | 用電度數 | 225 |
| c | 電費金額 | 368 |

### 期別代碼對照

| 期別代碼 | 意義 |
|----------|------|
| 11301 | 113年1月 |
| 11303 | 113年3月 |
| 11305 | 113年5月 |
| 11307 | 113年7月 |
| 11309 | 113年9月 |
| 11311 | 113年11月 |

## 3.3 視圖模式

### 依年度模式

- 合併同一年度的所有期別
- 顯示每年用電/電費總計
- 適合年度趨勢分析

### 依期別(每2月)模式

- 顯示每期用電/電費柱狀圖
- X軸為期別，Y軸為數值
- 適合單月份對比

### 指定期別各電號模式

- 顯示該期各電號的用電比較表
- 適合跨電號分析

## 3.4 資料統計

- 共有 14 組電號
- 涵蓋 9 個期別（11301 ~ 11503）
- 合計 126 筆記錄

# 6. 部署與維護

## 6.1 Git Hook 自動同步

每次 workspace commit 後，Hook 自動執行：

```bash
#!/bin/bash
cd /tmp/inventory
cp ~/.openclaw/workspace/inventory.json .
git add -A
git commit -m "Auto-sync"
git push origin main --force
```

## 6.2 LINE Bot Webhook 更新

當 ngrok 重啟後，URL 會改變，需要更新：

1. 前往 LINE Developers Console
2. 進入你的 Channel
3. Messaging API settings
4. 更新 Webhook URL
5. 點擊 Verify

## 6.3 常見問題與解決方式

| 問題 | 解決方式 |
|------|----------|
| 網頁 404 | 等待 GitHub Pages 部署（15-30秒） |
| LINE Bot 無回應 | 檢查 ngrok 是否運行 |
| 資料不同步 | 手動觸發 Git Hook |
| 條碼掃描失敗 | 確保光線充足，相機權權限開啟 |

# 7. 操作指南

## 7.1 庫存系統 - LINE 操作

### 上傳庫存照片

1. 打開 LINE，進入 Bot 對話
2. 拍攝商品照片（盡量清晰）
3. 傳照片並說「**我要盤點**」
4. Bot 回覆「已更新庫存，共 X 項商品」

### 查看結果

1. 開啟瀏覽器
2. 前往 https://jhetsai.github.io/inventory/
3. 查看「本次上傳品項」

## 7.2 庫存系統 - 網頁操作

### 搜尋商品

1. 在搜尋框輸入品號或品名
2. 系統即時過濾顯示結果

### 條碼掃描查詢

1. 點擊「📷 查詢」
2. 允許相機權限
3. 對準條碼掃描
4. 顯示商品完整資料

### 確認盤點

1. 點擊「✏️ 確認」
2. 勾選已盤點的商品
3. 可輸入備註
4. 點擊「✓ 完成」

## 7.3 台電系統 - 操作

### 查看用電概況

1. 開啟 https://jhetsai.github.io/openclaw/electricity_final.html
2. 預設顯示全部電號的總計

### 篩選特定電號

1. 在第一個下拉選單選擇電號
2. 圖表和資料自動更新

### 切換視圖

1. 「依年度」：顯示每年用電總計
2. 「依期別(每2月)」：顯示每期用電柱狀圖
3. 「指定期別各電號」：顯示該期各電號比較表
