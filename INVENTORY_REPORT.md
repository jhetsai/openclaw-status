# 3. 資料規格

## 3.1 資料檔案結構

```json
{
  "last_updated": "2026-04-04 13:55",
  "categories": [
    {
      "name": "分類名稱",
      "items": [
        {
          "code": "品號",
          "name": "品名",
          "quantity": 實際庫存,
          "onlineQuantity": 網購庫存,
          "verified": 是否確認,
          "note": "備註",
          "lastUploaded": "最後上傳時間"
        }
      ]
    }
  ]
}
```

## 3.2 欄位規格

| 欄位名 | 資料型態 | 必填 | 說明 |
|--------|----------|------|------|
| last_updated | string | 是 | 資料最後更新時間（YYYY-MM-DD HH:MM） |
| categories | array | 是 | 商品分類陣列 |
| code | string | 是 | 商品品號（條碼），最大12位數 |
| name | string | 是 | 商品名稱 |
| quantity | number | 是 | 實際庫存數量 |
| onlineQuantity | number | 是 | 網購庫存數量 |
| verified | boolean | 否 | 是否已確認（預設 false） |
| note | string | 否 | 備註文字 |
| lastUploaded | string | 否 | 最後上傳時間 |

## 3.3 商業邏輯規則

| 規則 | 邏輯 |
|------|------|
| 零庫存判定 | 當 quantity=0 且 onlineQuantity=0 |
| 零庫存行為 | 自動勾選（verified=true）並鎖定（disabled） |
| 最近上傳判定 | lastUploaded 時間在 24 小時內 |
| 顯示邏輯 | 只顯示最近上傳的品項 |

## 4.2 欄位寬度規格

| 欄位 | 寬度 | 對齊 | 特殊屬性 |
|------|------|------|----------|
| 品號 | 95px | 置中 | 不換行（nowrap） |
| 品名 | 100px | 左對齊 | 自動換行 |
| 實際 | 35px | 置中 | - |
| 網購 | 35px | 置中 | - |
| ✓ | 30px | 置中 | checkbox |
| 備註 | auto | 左對齊 | 可輸入文字 |

## 4.3 功能模組規格

### 統計看板
- 顯示 5 個統計數值卡片
- 即時計算總計
- 響應式設計（手機 2 欄、桌面 4 欄）

### 搜尋功能
- 即時過濾（oninput 事件）
- 支援品號和品名模糊搜尋
- 大小寫不敏感

### 條碼掃描
- 使用 Html5Qrcode 類別
- 解析度 fps=10, qrbox=250x150
- 支援後鏡頭（facingMode: environment）
- 掃描成功後自動查詢並顯示 Modal

### 確認模式
- 切換按鈕：✏️ 確認 ↔ ✓ 完成
- 可勾選已盤點商品
- 可輸入備註
- 離開編輯模式時自動儲存

| 類型 | 觸發條件 | 處理方式 |


# 9. 建議事項

## 9.1 安全性建議

1. **LINE Bot Token**：定期更換 Channel Access Token
2. **GitHub Token**：使用 Fine-grained tokens 並設定過期時間
3. **ngrok URL**：每次重啟後確認 Webhook URL 是否正確

## 9.2 備份策略

1. **inventory.json**：每次 LINE 更新後自動同步至 GitHub
2. **本地備份**：建議每週手動備份一次
3. **版本控制**：Git 已記錄所有歷史版本

## 9.3 監控建議

1. **GitHub Actions**：查看部署日誌確認是否成功
2. **LINE Developers Console**：確認 Bot 狀態和用量
3. **OpenClaw**：監控對話紀錄，確認 Bot 正常運作
