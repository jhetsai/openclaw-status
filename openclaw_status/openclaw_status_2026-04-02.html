5. 常用指令

5.1 OpenClaw 基本指令

# 查看狀態
openclaw status

# 查看幫助
openclaw help

# 啟動 Gateway
openclaw gateway start

# 停止 Gateway
openclaw gateway stop

# 重啟 Gateway
openclaw gateway restart

# 設定 Telegram Bot Token
openclaw config set channels.telegram.botToken "YOUR_TOKEN"

# 設定 Brave Search API Key
openclaw config set plugins.entries.brave.config.webSearch.apiKey "YOUR_KEY"

5.2 Cron 排程指令

# 查看排程
openclaw cron list

# 新增排程
openclaw cron add "30 8 * * *" --message "你的任務"

# 刪除排程
openclaw cron delete JOB_ID

# 手動執行排程
openclaw cron run JOB_ID

5.3 查詢與修改設定

# 查看設定檔位置
openclaw config file

# 設定值
openclaw config set KEY VALUE

# 取得值
openclaw config get KEY

# 刪除值
openclaw config unset KEY

6. 備份與還原

6.1 自動備份

每日 19:00 系統會自動產生備份，位於：

~/.openclaw/backups/YYYY-MM-DD/

6.2 手動備份指令

# 備份設定檔
cp ~/.openclaw/openclaw.json ~/.openclaw/backups/openclaw.json.$(date +%Y%m%d)

# 備份排程
cp ~/.openclaw/cron/jobs.json ~/.openclaw/backups/jobs.json.$(date +%Y%m%d)

# 備份記憶檔案
cp -r ~/.openclaw/workspace/memory/ ~/.openclaw/backups/memory.$(date +%Y%m%d)

6.3 還原指令

# 還原設定檔
cp ~/.openclaw/backups/openclaw.json.YYYYMMDD ~/.openclaw/openclaw.json

# 還原排程
cp ~/.openclaw/backups/jobs.json.YYYYMMDD ~/.openclaw/cron/jobs.json

# 重啟 Gateway 生效
openclaw gateway restart

7. 安裝新功能的方式

7.1 安裝外掛

# 透過 npm 安裝（以 LINE 為例）
npm install -g @openclaw/line

# 或在 openclaw.json 中設定
{
  "plugins": {
    "entries": {
      "line": {
        "enabled": true,
        "config": {}
      }
    }
  }
}

7.2 安裝 Skills（技能）

# Skills 放在 ~/.openclaw/skills/ 目錄下
# 每個 skill 是一個資料夾，包含 SKILL.md 說明檔

# 建議的 Skills：
# - weather：天氣查詢
# - node-connect：節點連線診斷
# - healthcheck：系統健康檢查

7.3 新增排程任務

# 透過指令建立
openclaw cron add "0 9 * * *" --message "每日任務描述" --delivery telegram --to 1181571031

# 或手動編輯 jobs.json
{
  "id": "unique-job-id",
  "name": "任務名稱",
  "enabled": true,
  "schedule": {
    "kind": "cron",
    "expr": "0 9 * * *",
    "tz": "Asia/Taipei"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "任務內容",
    "timeoutSeconds": 180
  },
  "delivery": {
    "mode": "announce",
    "channel": "telegram",
    "to": "1181571031"
  }
}

8. 疑難排解

8.1 常見問題

問題解決方式
Gateway 無法啟動檢查設定檔語法：openclaw config file
Telegram 無法收發確認 Bot Token 正確、網路連線正常
LINE Webhook 失敗確認 ngrok URL 已更新、WEBHOOK 設定正確
排程 Timeout增加 timeoutSeconds 數值（目前已設為 300 秒）
Brave Search 失敗確認 API Key 有效，檢查用量是否達上限

8.2 日誌查看

# 查看 Gateway 日誌
tail -f ~/.openclaw/logs/gateway.log

# 查看設定變更記錄
cat ~/.openclaw/logs/config-audit.jsonl | tail -20
